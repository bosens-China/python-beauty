## 1. 不同的读者，不同的路径

### 如果你是编程新手 🐣

- **建议顺序**：请按章节顺序**线性阅读**。
- **重点关注**：第一部分（语言基础）和第二部分（控制流）。不要急着去理解“元编程”或“并发”，先把基础打牢。
- **必做之事**：每一章的代码，请务必亲手敲一遍，而不是复制粘贴。手指的肌肉记忆会帮助你理解语法。

> 💡 **为什么手敲代码很重要？**  
> 编程不仅是“知道怎么做”，更是“熟练怎么做”。手敲代码能让你暴露拼写错误、缩进问题等细节，这些在复制粘贴时会被隐藏，但恰恰是初学者最需要掌握的基本功。

### 如果你是转语言的开发者 (TS/Java/Go) 🦅

- **建议顺序**：你可以**跳跃阅读**。
- **重点关注**：
  - **第二部分（控制流）**：关注 `match case` 和 EAFP（Easier to Ask for Forgiveness than Permission）异常处理哲学。
  - **第四部分（抽象）**：这是 Python 与静态语言差异最大的地方（如协议、装饰器）。
  - **第五部分（边界）**：关注 `Pydantic` 如何处理你熟悉的类型验证问题。
  - **第六部分（并发）**：关注 `asyncio` 与 Node.js Event Loop 的异同。
- **特别留意**：书中的 `::: info TS 开发者便签` 是专门为你准备的“翻译官”，它能帮你快速建立心智映射。

## 4. 关于类型注解 (Type Hints)

你会发现，本书中的几乎所有代码示例，都带有**类型注解**（type hints）：

```python
# 传统的教程可能这样写
def greet(name):
    return "Hello " + name

# 本书会这样写
def greet(name: str) -> str:
    return f"Hello {name}"
```

刚开始你可能会觉得繁琐。但请相信，这是现代 Python 工程的基石。

> 🔍 **什么是类型注解？**  
> 类型注解是在函数参数和返回值后添加的 `: 类型` 和 `-> 类型` 标记，用于说明预期的数据类型。例如 `name: str` 表示 `name` 应该是一个字符串，`-> str` 表示函数返回一个字符串。  
> **重要提示**：Python 在运行时**不会强制检查**这些类型（除非你使用额外工具如 `mypy`）。它们主要用于开发工具（如 VS Code、PyCharm）提供智能提示、错误检测和重构支持。

即使你是初学者，我也建议你从第一天开始，就尝试加上类型注解。这会让你养成极好的编码习惯，并显著提升代码可读性和可维护性。
