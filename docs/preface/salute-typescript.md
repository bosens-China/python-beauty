# ✉️ 给 TypeScript 开发者的一封信

> **"A language does not merely change how you write code; it changes how you think about problems."**
>
> **“一门语言改变的不只是你写代码的方式，更是你理解问题的方式。”**
>
> — _Anonymous_

---

如果你正在阅读这本书，很可能已经在 TypeScript 的世界里待了不短的时间。

你熟悉类型系统带来的安全感，习惯了清晰的接口边界，也体会过 V8 引擎在性能上的锋利。现在，你正站在 Python 的门口——这并不是一次逃离，而是一次视角的转换。

在这里，没有 `node_modules` 的体量焦虑，也很少需要为构建工具耗费心力。你依然会遇到错误，只是它们换了名字；你依然需要做工程决策，只是权衡的重心有所不同。

Python 的魅力，往往体现在那些不动声色的设计里：

- `with` 语句让资源管理回归语义本身；
- 列表推导式用更少的结构表达更多的意图；
- 从 `FastAPI` 到 `PyTorch`，生态本身在不断指向新的问题域。

如果你刚从 TypeScript 迁移而来，一个常见的误区，是试图在 Python 中复刻原有的写法与抽象。与其如此，不如先理解它所鼓励的风格：  
有些结构不必显式声明，有些约束更适合留给约定与文档，有些灵活性是刻意保留的设计选择。

**这正是 Python 的哲学所在：**  
追求可读性而非炫技，重视清晰而非完备，在工程可靠性与实践效率之间保持平衡。

像诗人一样写代码（Readable），  
像工程师一样构建系统（Robust），  
像问题解决者一样做出取舍（Pragmatic）。

希望这本书，能陪你完成一次克制而愉快的语言迁移。
